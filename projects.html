Drawing Website

The idea for this project came to me during online classes when I realized just how difficult it is to effectively communicate concepts that require equations and drawings over a video call. This is especially true for non-touchscreen devices, such as my Mac, because drawing with a touchpad can be tedious and awkward. So I set off to fix this issue for myself, with the end result shown on the left.

The main premise of the writing mechanics is to still use the touchpad but instead of pressing down on it to start drawing, you instead hold down the ALT or CTRL key. Doing this, alongside creating a makeshift stylus (made from attaching aluminium foil to a pencil), allows for a much more fluid writing experience.

I wrote the HTML and Javascript code from scratch (with help from Stack Exchange). As I grew more comfortable with HTML/Javascript, I continued to add features that I thought would be practical to have. As such it has advanced to the point that I have used it in place of paper notes.

Want to try it yourself? Click the button below! A warning that it is significantly easier to use with the stylus, and that it may take time to get comfortable with it. Best results are on Chrome.

3D Printer

3D printers are amazing. They allow you to design almost anything for a reasonable cost. The way it combines electronics, mechanical systems, and coding into one project is what inspired me to create my own 3D printer.

I based my project off of the Ultimaker 3D printer as I have worked with it in the past. My printer uses PLA as the printing material as it doesn't require a heated build plate. Two motors control the motion along the X and Y axis, the third motor drives motion in the Z direction, and the fourth motor is used to extrude the material into the print head.

The design of the frame is from scratch, as is the Arduino code that parses the G-code file. The wiring was done by referring to online articles that explain how the motor driver works.

This build is still in progress, stay tuned for updates on this!

Memory Pattern Alarm Clock

I have found that with enough practice, any alarm clock can be turned off without fully waking an individual up. The act of just pressing "Done" or "Snooze" may not be stimulating enough to get the person out of bed and ready to take on the day. That is where my idea for this alarm clock came from.

It starts off with the classic high pitched buzzing sound for an alarm clock, but turning the alarm off needs to be earned. To begin, you push the "Stop" button which ends the buzzing sound. Then you must complete 4 rounds of patterns. Each round works as follows: The clock generates a random sequence of 6 flashes that are displayed on 3 LED displays (for example the LED's flash in the order: Left, Left, Middle, Left, Right, Middle). Once this pattern is shown, the user must repeat the pattern by inputting it through 3 push buttons. Do this correctly and the alarm turns off. But be warned that if you stop halfway through and go back to bed, the alarm will activate again in 1 minute.

This device runs off an Arduino Uno, using a shift register, real time clock (RTC), and a piezo buzzer. The code was written from scratch. It has features such as turning the display on/off, setting the time and alarm time, and turning the alarm on/off. Currently the wire management leaves much to be desired.

Group Restaurant Selector

How many times have you been with a group of friends wanting to go grab some food but no one is decided where they want to go? What if there was an app for that? This is the inspiration for the Group Restaurant Selector Web App, a group project in my third year projects course.

Our idea was essentially Tinder for restaurants. It works as follows. One person will create a group on the app where they select parameters such as location and maximum travel radius. After this is inputted, a random code is generated. This code is then shared with everyone else who wants to participate. Every participant receives the same 5 randomly selected restaurants and votes yes or no for them. Once everyone is done, the results are presented to everyone. Not happy with any of the choices? You can get another 5 options by clicking "another round".

For this project, I helped create the backend of the web app using NodeJS and Express. It was a great introduction into backend development. I helped to create the structure of the web app using Express. I also worked on creating a data base in MySQL, updating it with user info, and fetching data from it to present on the web app. In addition, I helped set up fetch calls to Google API's to obtain restaurant data.

Space Invaders Game

A classic arcade game that has stood the test of time. I though it would be fun to try my hand in video game production and making a simple controller to play the game. What you see is the end result.

The game is run on an Arduino Uno, with the screen being a 8x8 LED display. The ship is controlled using a joystick and you fire using a push button. The entire game was written from scratch. Some challenges of programming this was having to keep track of all the different objects and how they interact with each other.

Prototype Lightsaber

Reason for the project? Lightsabers are arguably the coolest movie item ever and it would be fun to try and make my own.

The lightsaber is made using an Arduino Uno, an accelerometer, a small speaker and amplifier circuit, some LED's, and a microUSB reader. The frame of the lightsaber is made from cardboard (not to be used in wet battle conditions) and a string of green LED's provide the lights. When activated and deactivated, the LED's turn on/off in sequential order, giving the effect of the blade expanding/shrinking. In addition, the speaker produces the corresponding sound of ignition/shutoff and other sounds are played depending on the readings from the accelerometer.

While the length of the blade makes it seem more like a "lightdagger", I think it's a very cool proof of concept and I plan on making a version 2 sometime in the future!

Line Following Robot

This build is what started my passion for Arduino projects. The Line Following Robot was part of my second year projects courses and it really opened my eyes to the wonders of electronics.

An Arduino Uno operates the robot (named Wall-E) and two brushed DC motors propel it on the track. An array of 5 IR sensors detect where the black lines are and send the data back to the board to change direction. The code for this project was written from scratch, and the wiring was done by referencing class notes.
